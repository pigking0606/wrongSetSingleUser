# AI 解题管道 Prompt 编写规范

本项目的 AI 分析系统通过 **5 层管道** 处理用户上传的错题图片，每层有独立的 prompt 设计策略。

---

## 管道总览

```
图片 → [视觉模型] → JSON解析 → [去重模型] → [L1包裹] → [L2 LaTeX修复] → [L3正则清理] → 入库
```

每层调用一个 AI 模型（或正则处理），prompt 按其角色分为三类：

| 类别 | 模型类型 | 用途 | 当前模型 |
|------|---------|------|---------|
| 视觉识别 | vision (图片理解) | OCR + 分类 + 解题 | qwen-vl-plus |
| 文本处理 | text (文本理解) | 去重、LaTeX修复、建议 | deepseek-chat / qwen-plus |
| 格式化 | 无 (纯正则) | LaTeX 包裹和清理 | — |

---

## 一、视觉模型 Prompt（最关键）

**位置**：`src/lib/ai.ts` — `buildSystemPrompt()` + user message
**模型**：qwen-vl-plus / qwen-vl-max
**参数**：temperature=0, max_tokens=8192, response_format=json_object

### 1.1 System Prompt 结构

```
【身份定义】你是考研命题专家。
【知识约束】已知科目体系如下，classification 必须从中选取：
  【数据结构】
    线性表：定义与基本操作、顺序存储、链式存储
    栈和队列：栈的定义、队列的定义、栈和队列的应用
    ...

【输出格式】JSON（不含 markdown 包裹）：
{"ocrText":"...", "questionType":"...", "classification":{...}, ...}
```

**设计原则**：
1. **身份先行** — "你是考研命题专家" 比 "你是一个AI助手" 更能引导专业输出
2. **白名单约束** — 给出完整科目树，AI 只能从中选，不会自创分类
3. **格式先行** — system prompt 就给 JSON schema，AI 知道目标结构
4. **简短有力** — system prompt 控制在合理长度（过长会被截断或稀释注意力）

### 1.2 User Message 结构（最重要）

这是真正驱动 AI 行为的地方，**占 prompt 效果的 70%+**。

**结构模板**：
```
请分析这道[错题/题目]。

严格要求：
1. ocrText：【OCR规范】
   - 去掉题号前缀
   - 忽略手写笔迹（只识别印刷体）
   - 选择题选项每行一个

2. classification：【分类规范】
   - 必须从已知科目体系中选，不得自创

3. correctAnswer：【答案规范】
   - 只给出正确答案

4. explanation：【解析规范】
   - 至少200字
   - 必须包含：①知识点回顾 ②分步解题 ③易错点提醒
   - 【数学公式规范】所有公式包裹在$...$中，禁止拆开
   - 上下标必须花括号，分数用\\frac

5. solutions：【多解法规范】
   - 至少2种解法
   - 每种含 name + steps[] + answer

6.【关键】JSON 反斜杠规范：
   - 正确：$\\\\frac{1}{2}$  $\\\\lim_{x \\\\to 0}$
   - 错误：$\\frac{1}{2}$（破坏 JSON）
```

### 1.3 经过验证的有效约束

以下是经过 80+ 道题验证后保留的关键约束，**绝对不能删除**：

```
✓ 去掉题号前缀（如'32.'、'【2021统考真题】'）
  → 原因：AI 会把题号识别进题干，导致题干显示"32. 下列关于..."

✓ 忽略手写笔迹（手写答案/演算/批注一律不识别到ocrText中）
  → 原因：用户拍的照片上有手写答案，AI 会混入 OCR

✓ 行列式/矩阵必须识别为整体 LaTeX 表达式，禁止拆成多行
  → 原因：AI 会把 3x3 矩阵拆成 3 行独立文本

✓ $...$ 完整包裹，禁止写成 $a = $b 形式
  → 原因：AI 喜欢碎片化数学表达式，KaTeX 无法渲染

✓ ^{...} 和 _{...} 内部绝对不能有 $ 符号
  → 原因：AI 会写出 $x^{$...$}$ 导致 KaTeX 崩溃

✓ JSON 内所有 LaTeX 反斜杠写成双反斜杠 \\\\
  → 原因：JSON 中 \f \l \t 是转义字符，单反斜杠会破坏解析
```

### 1.4 经历过失败的约束（反面教训）

```
✗ "请用中文输出" — 太弱，AI 对"中文"的理解不一致
  → 应该："ocrText 必须完整逐字识别，保持原文语言"

✗ "解析要详细" — 太模糊
  → 应该："至少200字，包含①知识点回顾②分步解题③易错点提醒"

✗ 没有写"忽略手写笔迹"
  → 后果：用户照片上的手写答案被识别进题干

✗ "OCR识别题目"（没写具体规范）
  → 后果：题号、页码、页眉页脚全部被识别

✗ 没强调 JSON 反斜杠规范
  → 后果：AI 返回 \frac 但 JSON.parse 把它解析成换页符 \f + "rac"
```

## 二、文本模型 Prompt

### 2.1 去重 Prompt（DEDUP）

**位置**：`src/lib/ai.ts` — `DEDUP_PROMPT`
**模型**：文本模型（qwen-plus / deepseek-chat）
**参数**：temperature=0, max_tokens=4096

```
你是文本精简助手。AI生成的文本可能反复推翻自己的说法。

任务：删掉所有"自我推翻"的内容（如"等等，不对，应该重新考虑..."），只保留最终解答。

规则：
1. 删除推翻前文的部分，只保留最后确定的结论
2. 不改变最终结论的任何内容
3. 如果没有任何推翻，原样返回
4. 绝不新增任何内容
5. 直接返回精简后文本，不要解释
```

**设计原则**：
- **只删不改** — 限制 AI 只做删除，防止它"优化"正确答案
- **原样返回兜底** — 明确告知"如果没有问题 → 原样返回"，防止 AI 强行修改
- **结构化输入** — 用【字段名】标记分隔不同字段，输出用正则提取

### 2.2 LaTeX 修复 Prompt（LATEX_FIXER）

**位置**：`src/lib/ai.ts` — `LATEX_FIXER_PROMPT`
**模型**：文本模型
**参数**：temperature=0, max_tokens=8192

```
你是 LaTeX 格式化专家。修复 JSON 中每个字段的 LaTeX 格式错误。

严格规则：
1. 每个完整数学表达式必须包裹在一个 $...$ 中，禁止拆成 $\ln$ $y$
2. ^{} 和 _{} 内部绝对不能出现 $ 符号
3. \left 和 \right 必须成对出现在同一个 $...$ 内
4. $ 必须成对出现
5. 所有 LaTeX 命令必须在 $...$ 内部
6. 只修复 LaTeX 格式，不改变题目含义

输出：直接返回修复后的 JSON，字段结构与输入一致，不要添加任何解释。
```

**设计原则**：
- **具体到反例** — 每个规则都给出错误示例（"\$\\ln\$ \$y\$"），AI 更容易理解
- **输出格式明确** — "返回 JSON，不要解释"，防止 AI 多话
- **容错设计** — 如果修复失败，代码直接返回原始文本（不抛异常）
- **输入格式化** — 用 `JSON.stringify(texts, null, 2)` 传入，AI 能理解结构

### 2.3 重解析 Prompt（REANALYZE）

**位置**：`src/app/api/reanalyze/route.ts`
**模型**：视觉模型（full模式）/ 文本模型（answer模式）

```
你是考研命题专家。请重新分析以下题目，严格返回纯JSON。

{ JSON schema }

【ocrText规范】
- 去掉题号前缀
- 只保留印刷体题干，忽略手写笔迹
- 选择题选项每行一个

【数学公式规范】
- 完整公式一个$...$块包裹
- 行列式用 \begin{vmatrix}...\end{vmatrix}
- JSON 内反斜杠写成双反斜杠 \\\\
```

**重解析的特殊点**：
- full 模式必须**发送图片**（不要因为已有 OCR 就只发文本）
- 用户可提供 `reason` 参数，插入 user message 指导 AI 修正
- 有旧 OCR 文本作为参考："题目的参考文本（可能存在格式错误，以图片为准）"

## 三、Prompt 设计通用原则

### 3.1 结构化层次

```
system prompt：角色 + 约束 + 输出格式（简短，30% 权重）
user message： 具体任务 + 详细规范 + 反例（详细，70% 权重）
```

### 3.2 规范编写模式

每条规范遵循 **"应该 → 禁止 → 示例"** 三元结构：

```
【好】所有公式包裹在 $...$ 中，禁止 $a = $b 碎片形式，正确写法：$a = b$
【差】数学公式要写对
```

### 3.3 必须包含的元素

| 元素 | 示例 | 原因 |
|------|------|------|
| 身份 | "你是考研命题专家" | 激活专业领域知识 |
| 边界 | "只从已知科目体系中选择" | 防止 AI 编造 |
| 格式 | "返回纯JSON，不要markdown包裹" | 防止 ```json``` 包裹 |
| 反例 | "禁止 $a = $b 形式" | 比"要正确"更有效 |
| 示例 | "正确：$\\\\frac{1}{2}$" | 对齐 AI 输出格式 |
| 长度 | "至少200字" | 防止 AI 偷懒 |

### 3.4 绝对不能做的事

```
✗ 用"请"开头 — 弱语气导致 AI 不严肃对待约束
✗ 模糊描述 — "详细解析" → 必须量化"至少200字，包含①②③"
✗ 假设 AI 理解上下文 — 每层 prompt 独立完整，不依赖上一层
✗ 用 markdown 示例却不说明 — AI 会返回 markdown 包裹的 JSON
✗ temperature > 0 — 解题任务需要确定性，temperature=0
```

## 四、完整 Prompt 模板（可直接复用）

### 4.1 视觉识别 Prompt 模板

```
【System】
你是[领域]命题专家。已知分类体系如下，必须从中选取：

[分类树]

输出 JSON（不含 markdown 包裹）：
{"ocrText":"...","questionType":"...","classification":{"subject":"","chapter":"","knowledgePoint":""},"correctAnswer":"","explanation":"","solutions":[{"name":"","steps":[],"answer":""}],"confidence":0.95}

【User - 图片消息】
请分析这道题目。

严格要求：
1. ocrText：完整逐字识别，去掉题号前缀（如'32.'、'【2021真题】'），只保留印刷体题干。忽略手写笔迹。
2. classification：必须从已知科目体系中选，不得自创。
3. explanation：至少[200]字，包含①知识点回顾②分步解题③易错点提醒。
4. solutions：至少[2]种解法，每种含 name+steps[]+answer。
5. 【关键】JSON 内 LaTeX 反斜杠写成双反斜杠 \\\\，如 $\\\\frac{1}{2}$。

【数学公式规范】
- 完整公式一个 $...$ 块包裹，禁止 $a = $b 碎片
- ^{} 和 _{} 内部绝对不能有 $
- 上下标用花括号：x^{2} 非 x^2
- 矩阵用 \\begin{pmatrix}...\\end{pmatrix}
```

### 4.2 文本修复 Prompt 模板

```
【System】
你是[格式]格式化专家。修复 JSON 中每个字段的格式错误。

规则（每个规则给出正确+错误示例）：
1. [规则描述]，禁止[错误形式]，正确形式：[示例]
2. ...

只修复格式，不改变内容。返回修复后的 JSON，不要解释。

【User】
请修复以下 JSON 中每个字段的格式，返回相同结构的 JSON：

[格式化JSON输入]
```

### 4.3 建议类 Prompt 模板

```
【System】
你是[领域]规划助手。根据学生的学习情况，建议[N]个具体任务。

【User】
近期数据：
- 今日未完成任务：[...]
- 今日小结：[...]
- 历史任务模板：[...]
- 可用分类：[...]

要求：
- 返回纯JSON，不含markdown
- 未完成任务优先安排
- 任务具体到章节，如"完成线性代数第3章课后习题"
- 不要推荐宽泛任务（"复习数学"）

JSON格式：
{"tasks":[{"title":"...","chapter_id":N,"description":"...","difficulty":N}],"reason":"..."}
```

## 五、调优流程

当 AI 输出不符合预期时，按以下顺序排查：

```
1. AI 返回空 → 检查模型名是否有效、API Key是否过期
2. AI JSON 格式错误 → 检查 prompt 是否要求"不含 markdown 包裹"
3. AI 编造分类 → 检查 system prompt 是否给出完整分类树
4. LaTeX 渲染失败 → 检查是否经过 3 层管道（包裹→AI修复→正则清理）
5. OCR 包含手写 → 检查 user prompt 是否有"忽略手写笔迹"
6. 解析包含题号 → 检查 user prompt 是否有"去掉题号前缀"
7. AI 自我推翻（"等等不对，重新考虑"） → 检查是否经过去重管道
8. AI 偷懒（解析太短） → 量化要求"至少200字"
```
